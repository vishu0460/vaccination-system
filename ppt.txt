=== Slide 1: Project Title Slide ===
• VaxZone - Smart Vaccination Management Platform
• Secure Booking, Real-time Notifications & Certificate Management
• Developed by Professional Full-Stack Team

=== Slide 2: Introduction ===
• Problem Statement:
  - Manual vaccination booking leads to long queues and missed slots
  - Lack of real-time availability and notifications
  - No digital certificate verification or history tracking
  - Inefficient admin management of centers and drives
• Objective:
  - Build a production-ready platform for seamless vaccination scheduling
  - Enable citizens to book slots easily with real-time updates
  - Provide admin dashboard for analytics and operations
  - Generate verifiable digital certificates with QR codes

=== Slide 3: Project Overview ===
• Complete vaccination management ecosystem
• Citizens: Discover centers/drives, book slots, track bookings, verify certificates
• Admins: Manage centers, drives, slots, analytics dashboard with charts
• Key capabilities: Real-time slot availability, notifications (email/SMS/in-app), PWA support
• Target Users:
  - Citizens (primary): Age 5-100 for vaccination booking
  - Vaccination Center Admins
  - Super Admins for system-wide management

=== Slide 4: Key Features ===
• User Features:
  - Registration/Login with email verification & 2FA
  - Smart search for centers/drives by location/age
  - Real-time slot booking, cancel/reschedule
  - Vaccination history & digital certificates (QR/PDF)
  - Nearby centers discovery & reviews/ratings
  - In-app notifications & reminders
• Admin Features:
  - Dashboard with charts (bookings, slots, analytics)
  - CRUD: Centers, Drives, Slots, Users, Bookings
  - Feedback/Contact management & responses
  - Export bookings to CSV
  - Audit logs & search analytics
• Advanced:
  - PWA with offline support
  - JWT auth with refresh tokens

  - SEO optimized with sitemap/robots.txt

=== Slide 5: Technology Stack ===
• Frontend:
  - React 18 + Vite + React Router + Bootstrap 5
  - Chart.js/Recharts for admin analytics
  - Axios API client with auto token refresh
  - PWA: Service Worker + Manifest
  - jsPDF + QRCode for certificates
• Backend:
  - Spring Boot 3 + Spring Security + Spring Data JPA
  - JWT Authentication, Role-based access (USER/ADMIN/SUPER_ADMIN)
  - Flyway migrations + MySQL/H2
  - OpenAPI/Swagger documentation
• Infrastructure:
  - Docker Compose + Nginx reverse proxy
  - Multiple profiles: local/prod/dev
  - Health checks + graceful shutdown
  - SMTP (Gmail) + Twilio SMS integration
• Testing: JUnit/Mockito, Vitest/Playwright E2E

=== Slide 6: System Architecture ===
• Client → Nginx Reverse Proxy → Frontend (Static SPA) / Backend API
• SPA communicates with Backend REST APIs (/api/auth, /api/user, /api/admin)
• Backend: Controller → Service → Repository → Database
• Stateless JWT auth: Login → Access/Refresh tokens → Auto refresh
• Data Flow:
  1. User searches centers/drives (public APIs)
  2. Auth → Book slot → Notification queued
  3. Admin dashboard fetches analytics/bookings
  4. Certificate generation on booking completion (QR/PDF)
• Multi-profile: H2 (local), MySQL (prod), caching enabled

=== Slide 8: Database Design ===
• Core Entities & Relationships:
  - Users (1:M) → Bookings → Slots → Drives → Centers
  - Users (1:M) → Certificates ← Bookings
  - Users (1:M) → Feedback/Contacts/Notifications
  - AuditLogs track all actions
• Key Tables (20+):
  - users, roles, user_roles
  - vaccination_centers, vaccination_drives, slots, bookings
  - certificates, notifications, feedback, contacts
  - audit_logs, search_logs, rate_limits
• Features: Soft deletes, indexes for analytics, versioning for slots
• Migrations: 20+ Flyway scripts for schema evolution
• Seed data: Test users/centers/drives/news

=== Slide 9: API & Integration ===
• RESTful APIs: /api/public/* (drives/centers), /api/auth/* (login), /api/user/*, /api/admin/*
• Frontend Integration:
  - Axios client with interceptors (token refresh on 401)
  - DTOs for all requests/responses
  - Error boundaries & global error handling
• External Integrations:
  - SMTP: Email notifications/verification (Spring Mail)
  - Twilio SMS (optional)
  - PDF/QR: OpenPDF + ZXing for certificates
• Documentation: Swagger UI at /swagger-ui.html
• Security: CORS, rate limiting per client/IP

=== Slide 10: Security Features ===
• Authentication: JWT (access 15min/refresh 7days), BCrypt passwords
• Authorization: Role-based (@PreAuthorize), method-level security
• Protection:
  - Rate limiting (60/min per IP/client)
  - Brute-force lockout (5 attempts → 15min lock)
  - Custom JWT filter chain
  - Stateless sessions (no server-side session storage)
• Data Protection:
  - HTTPS ready (Nginx certs)
  - Input validation (@Valid DTOs)
  - Audit logging for all actions (user/IP/action/resource)
  - Soft deletes prevent accidental data loss
• Frontend: Protected routes, token storage in memory

=== Slide 11: Workflow / Process Flow ===
• User Journey:
  1. Register/Login → Email verify → Profile setup
  2. Search drives/centers → Filter by city/age/date
  3. Select drive/slot → Book → Instant confirmation
  4. Get reminder notifications → Visit center
  5. Admin marks complete → Certificate (QR/PDF) generated
• Admin Journey:
  1. Login → Dashboard overview (charts/stats)
  2. Manage drives/slots/bookings/users
  3. Respond feedback/contacts → Export reports
• Booking Flow: Real-time capacity update, unique user-slot constraint

=== Slide 13: Challenges & Solutions ===
• Challenge: Real-time slot availability
  - Solution: Atomic updates + capacity counters in DB
• Challenge: Scalable notifications
  - Solution: Background scheduler with retry (5 attempts), deduping
• Challenge: Certificate security
  - Solution: QR codes + digital verification codes, PDF tamper-proofing
• Challenge: Multi-tenant admin (center-specific)
  - Solution: Center-admin junction table + RBAC
• Challenge: Search performance
  - Solution: City indexes + search logs analytics
• Challenge: Deployment complexity
  - Solution: Docker Compose + profiles (local/prod)
